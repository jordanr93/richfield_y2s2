5.2.3 Review the System Design

At this point, it is helpful to revisit the tasks involved in creating the system design:

    •   Chapter 4: Focused on requirements modeling and functional decomposition
        diagrams (FDDs) to break down complex business operations into smaller,
        manageable functions.
    •   Chapter 5: Covered structured data and process modeling, including data flow
        diagrams (DFDs), and the creation of process descriptions for documenting business
        logic and processing requirements.
    •   Chapter 6: Focused on the object-oriented (OO) model of the new system, which
        included use case diagrams, class diagrams, sequence diagrams, state transition
        diagrams, and activity diagrams.
    •   Chapter 7: Addressed the selection of a development strategy.
    •   Chapter 8: Focused on designing the user interface.
    •   Chapter 9: Discussed data design issues, analyzing relationships between system
        entities, and creating entity-relationship diagrams (ERDs).
    •   Chapter 10: Covered considerations for overall system architecture.

Together, these tasks contributed to creating a comprehensive design and implementation
plan for the system.
