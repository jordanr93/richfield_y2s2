   5.8.7 Online Documentation


With many users preferring immediate help, online documentation has become increasingly
important. Context-sensitive help, on-screen demos, hints, and tips are expected by users and
are commonly found in popular software packages. For in-house systems, these features
should be part of the system requirements. If someone other than the analysts is developing
the documentation, they should be involved early on to familiarize themselves with the
software and begin creating the necessary materials.

Online documentation can reduce the need for IT staff to assist with issues through
telephone, email, or face-to-face support. Interactive tutorials, often using visual media, are
especially popular. For example, platforms like YouTube provide accessible tutorials, and
websites, intranet sites, and internet-based technical support can offer additional resources.

Though online documentation is essential, written documentation still holds value, especially
for training and reference purposes. Systems analysts or technical writers usually prepare the
manual, with input from users during the development process. The development of
documentation can take time, and it should be planned well in advance to ensure that a
complete package, including documentation, is ready for release once the software coding is
finished.
